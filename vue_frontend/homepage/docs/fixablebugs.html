Monster-specific bugs due to be fixed

  • 'Early' protons:

    • Monster is unable to assign a parent heavy atom to a proton, if the proton appears BEFORE the heavy atom in the PDB file.

    • The common example is that of terminal protons found at the beginning of the first residue, rather than after the last heavy atom in the same residue.

    • An implementation that derives all calculations and assignments residue-by-residue, rather than atom-by-atom, when reading in the ACF file, is planned.
