Auxiliary Tools in Felix 98.0

Suggest Name

This is a menu-driven macro/nawk script for assigning cross peaks in multi-dimensional NMR spectra. For a given cross peak in an nD spectrum, plausible assignments are presented for all dimensions. The search criterion is simply to match the coordinates of the cross peak with that of individual resonances that fall within a (user-specified) window of chemical shifts. Some salient features of this tool are:

  • 2D and 3D homonuclear as well as 13C/15N-edited heteronuclear spectra can be analyzed
  • Searches can be restricted to certain classes of spins (exchangeable, non-exchangeable, aromatic, aliphatic)
  • Aliasing can be accounted for in all dimensions
  • Assignments can (almost) always be entered into the DBA without typing (exception: ambiguous assignments)
  • Multiple rounds of searches can be performed with different search parameters and without altering the DBA

    • Suggest Name can be accessed from the Peaks menu in Felix 98.0. Obviously, you need a DBA file with the entity xpk:peaks already present. The script requires a chemical shift table that is identical in format to the one developed in the lab. A script file seq2tbl.s is available that will produce a chemical shift table (with dummy shifts) for any amino acid sequence. The actual chemical shifts can then be inserted manually or using nawk scripts or other programs. The table should be located in the directory specified by the &etcpfx symbol.

    Clicking on Suggest Name item will produce a cursor which enables selection of any cross peak in the display. A pop-up menu will then start-up to query the user for inputs. Note that the matrix needs to be always referenced in ppm (exception: D1 dimension of strip matrices). Also note that pre-existing assignments, if any, will be displayed in the window immediately below the results window. Click on this link to see an example pop-up menu with brief descriptions of functionalities.

    The options available for the various flags in the pop-up menu are summarized:

    • Covalent link
      • D1-D2
      • D2-D3 (3D matrices only)
      • D1-D3 (3D matrices only)
      • None (for homonuclear data)
    • Matrix type ( 2D matrices only)
      • Regular matrix
      • Strip matrix [atom subset automatically set to "Lock" in D1 dimension]
    • Nucleus
      • 1H
      • 13C
      • 15N
    • Atom subset
      • Lock
      • All
      • Exchangeable
      • Non-exchangeable
      • Aliphatic
      • Aromatic
    • Aliasing
      • Not aliased
      • Aliased (+1x) [aliased once downfield]
      • Aliased (-1x) [aliased once upfield]
      • Aliased (+2x) [aliased twice downfield]
      • Aliased (-2x) [aliased twice upfield]

    The tolerance parameter (in ppm) controls the width (+/- tolerance) of the search region in each dimension. Search results are posted in the results window. Assignments can be selected using the cursor; the selections appear in the window immediately below it. In case one prefers to enter the assignment by hand (e.g. when the assignment is ambiguous), they can be entered in this space. Update DBA will store the current assignments. Cancel will exit without any changes to the DBA while Run Search will conduct another search.
