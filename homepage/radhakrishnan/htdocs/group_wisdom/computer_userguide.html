User Guide

Computers Overview

The computers in the Radhakrishnan laboratory are split into two distinct groups easily defined by the rooms they are found in:

  1. The "Lab" group:

    • Maniatis (Windows NT)

    • Scopes (Windows NT)

    • Pauling (Windows 2000)

  2. The "Office" group:

    • Purcell (Windows 2000/RedHat Enterprise Linux)

    • Dirac (Windows 2000/RedHat Enterprise Linux)

    • Hilbert (Windows 2000/RedHat Enterprise Linux)

    • Overhauser (Windows 2000/RedHat Enterprise Linux)

    • Slichter (Windows 2000/RedHat Enterprise Linux)

    • Abragam (SGI)

*

The RADHAKRISHNAN Domain

The log-in screen for windows has an extra option called 'Log in to:' which allows a user to select either 'this computer' or 'RADHAKRISHNAN'.

The user will NOT be able to log in to 'this computer' as their username and password only exists in the RADHAKRISHNAN domain.

If you get a login error, please double check this option, to make sure it says 'RADHAKRISHNAN'.

*

The 'H:\' Drive on Windows

Logging in to the RADHAKRISHNAN domain automatically loads an extra disk drive called 'H:\'

If you look in 'My Computer' (Right click->explore) you'll see something like this:

"This drive is physically located on the file server that controls the RADHAKRISHNAN domain."

This file server is automatically backed up on a regular basis, therefore it is important that you save all your precious data in the 'H:\' drive.

*

Roaming Profiles

When a user logs in for the first time, the file server automatically creates a new roaming profile for that user.

All the computers in the Office group will DELETE the profile when you log out, because it is saved on the file server.

Therefore it is recommended that all files are saved in 'H:\'.

Furthermore it is HIGHLY RECOMMENDED, that all users MOVE their 'My Documents' directory to 'H:\'.

Here's how:

  • Right click on your 'My Documents' folder on the desktop, and click on 'properties'.

  • In the first tab ('Target') click on the 'Move' button, and then type 'H:\', and press return.

  • A pop-up will appear asking you if you want to move all the associated files, choose 'Yes'

*

Overview of Computers in the 'Lab' Group

Each computer in the Lab group has it's own unique username, which is the same name as the computer's name. (e.g.: the username for Scopes is 'scopes').

Never log on with your own username on these computers because you will not be able to access the data.

Again, always log on into the RADHAKRISHNAN domain, with the sole exception of Pauling (see below)

All data is saved (if possible) on the H:\ drive, which will only appear if you log in with the correct username.

Please DO NOT save data anywhere else, if possible.

Please DO NOT use these computers for personal use, including e-mail and/or internet.

Pauling/UV Solutions Complication

UV Solutions has a bug that disables it from recognising the permissions of network users.

Therefore, a local 'pauling' account is used instead (log onto 'Pauling (This computer)' instead of RADHAKRISHNAN).

It is imperative that the user does NOT try to log onto the RADHAKRISHNAN domain using the user 'pauling'.

The one main implication is that the profile is no longer saved onto monster. So do NOT save any files/folders anywhere other than to H:\.

*

Accessing Other Lab Members' data

One of the advantages of the windows network set-up is the ability to access anyone's data from any computer in the lab.

In order to do this:

  • Right-click on 'My Computer' and choose 'Map network drive'.

  • in the drive path, type: '\\monster\user' where user is the username of the person whose data you want to access.

  • Press 'Finish'.

It should be noted that you will only have read permissions on the newly mapped drive and will not be able to edit anyone else's data (thank goodness).
